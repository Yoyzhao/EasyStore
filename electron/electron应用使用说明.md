# EasyStore Windows 客户端使用说明

欢迎使用 EasyStore 桌面应用！本程序已将前端界面与后端服务整合为一个独立的 Windows 应用程序。

## 1. 应用说明
EasyStore 客户端采用 Electron 框架打包，内部集成了 Python 后端服务。运行 `EasyStore.exe` 时，程序会自动启动后台服务并打开管理界面，无需手动配置 Python 环境或启动服务器。

## 2. 安装与启动

### 免安装版 (Portable)
1. **获取程序**: 找到打包生成的目录 `d:\Workspace\EasyStore\electron\release_final\`。
2. **单文件版**: 直接运行 **`EasyStore 1.0.0.exe`**。这是单文件免安装版，首次运行会自动解压到临时目录并启动。
3. **压缩包版**: 解压 **`EasyStore-1.0.0-win.zip`**，运行其中的 `EasyStore.exe`。
4. **部署**: 您也可以直接拷贝 `win-unpacked` 文件夹到任意位置运行。

## 3. 数据存储与备份

### 默认存储位置
程序启动后，默认会在 `EasyStore.exe` 所在的同级目录下创建一个 `data/` 文件夹：
- **数据库**: `data/db/easystore.db`
- **上传文件**: `data/uploads/`
- **系统配置**: `data/settings.json`

### 自定义数据路径 (重要)
如果您希望将数据存储在其他盘符（如 `E:\EasyStoreData`），请按以下步骤操作：
1. 打开应用，进入 **系统设置**。
2. 在 **数据保存路径** 中输入新的绝对路径（例如 `E:\EasyStoreData`）。
3. 保存设置。
4. **手动迁移**: 关闭程序，将原 `data/` 目录下的所有子文件夹（`db`, `uploads`）手动移动到新路径下。
5. 重新启动程序。

## 4. 常见问题

### Q1: 启动时提示“等待后台服务启动...”时间过长？
- 我们已将后端模式优化为 `onedir`，极大提升了二次启动速度。
- 首次运行或防火墙拦截可能导致启动变慢，请检查是否有防火墙弹窗提示，并选择“允许访问”。

### Q2: 软件关闭后后台服务还会运行吗？
- 不会。EasyStore 客户端在关闭所有窗口时，会自动安全终止后台的 Python 服务进程。

### Q3: 登录提示“网络异常”？
- 请确保后端服务已成功启动（通常在启动 1-3 秒后即可连通）。
- 程序内部已自动处理 Electron 环境下的 API 地址转发（指向 `http://localhost:8000`）。

### Q4: 如何备份数据？
- 您可以直接进入 **系统设置** -> **数据备份** 进行备份。
- 或者直接手动拷贝整个 `data/` 文件夹（或您自定义的数据路径文件夹）到安全的位置。

## 5. 开发与打包注意事项
- **后端路径**: 核心后端程序位于 `resources/bin/easystore-server/easystore-server.exe`。
- **图标配置**: 应用图标文件为 `electron/icon.ico`，打包配置已在 `package.json` 中固化。
- **打包输出**: 当前打包输出目录已调整为 `release_final/` 以避免文件锁定冲突。
- **权限建议**: 建议将应用文件夹放在具有读写权限的盘符下，避免放在 `C:\Program Files` 等受保护目录（除非以管理员身份运行）。
